| Token | Hex | Usage |

### Theme Palettes (from THEMES constant)

Applied dynamically based on wish's selected theme. The theme colors override gift box accents, confetti particle colors, and ambient glow.

| Theme ID | Primary | Secondary | Surface |
|----------|---------|-----------|---------|
| sunrise | `#ff6f59` | `#ffb84d` | `#fff8ef` |
| ocean | `#4d96ff` | `#6bcb77` | `#e8f4f8` |
| lavender | `#c44dff` | `#ff6b9d` | `#f8f0ff` |
| forest | `#2bb39c` | `#6bcb77` | `#f0faf5` |
| midnight | `#4d4d7a` | `#c44dff` | `#1a1a2e` |

### 60/30/10 Distribution

- **Dominant (60%):** Theme surface color as page background. `--cream` (#fff8ef) default for sunrise theme.
- **Secondary (30%):** Gift box body/lid use theme primary color. Text areas and content zones are unstyled (inherit background).
- **Accent (10%):** Theme primary and secondary colors reserved for: gift box accents, glow radial gradient, confetti particle colors, sender name color, active progress dots.

### Accent Reserved For (explicit list)

1. Gift box lid and ribbon accents (theme primary)
2. Gift box glow radial gradient (theme secondary at 30% opacity)
3. Sender name text color (theme primary via `--coral` override)
4. Active sentence progress dots (theme primary)
5. Confetti explosion particle colors (theme primary + secondary)
6. "Happy Birthday!" finale text (theme primary)
7. Replay button background (uses `--mint` -- stays constant across themes)
8. Pause/resume button (uses `--mint` -- stays constant across themes)

### Destructive Color

Not applicable for this phase. No destructive actions exist (no delete, no undo, no confirmation dialogs).

### Accessibility Note

Midnight theme has dark surface (`#1a1a2e`). When midnight is active, text color must invert to light (white or near-white) to maintain 4.5:1 contrast ratio. All other themes use dark text on light surfaces.

Source: `ThemeSelector.jsx` THEMES constant, `index.css` `:root` variables.

## Animation Contract

All animation code MUST follow `.claude/skills/framer-motion-patterns/SKILL.md`. Key contracts:

### State Machine Phases

```
IDLE -> GIFT_BOX -> UNWRAPPING -> SENTENCE -> FINALE -> PHOTOS -> COMPLETE
                                                              |
                                                     (replay) -> IDLE
```

### Timing Contracts

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Gift box mount | 400ms | spring (stiffness 200, damping 20) | Page load after data fetch |
| Gift box split | 1000ms | ease-out | Tap on gift box |
| Particle burst | 800ms | ease-out | Overlaps with split start |
| Sentence typewriter | 40ms per character | linear | Auto-start on phase enter |
| Sentence fade-in | 300ms | ease-out | Phase transition |
| Sentence fade-out | 200ms | ease-in | Tap to advance |
| Photo crossfade | 500ms | ease-in-out | Auto-advance (3-4s timer) or manual nav |
| Confetti explosion | 3000ms | N/A (physics) | After last sentence |
| Happy Birthday text | 500ms | spring (stiffness 150, damping 15) | 500ms after confetti start |
| Sparkle effects | 2000ms | ease-out | Overlaps with confetti |
| Replay button fade-in | 400ms | ease-out | After photo slideshow ends |

### AnimatePresence Key Pattern

```jsx
// One AnimatePresence mode="wait" in ExperienceOrchestrator
<AnimatePresence mode="wait">
  {state.phase === 'GIFT_BOX' && (
    <motion.div key={`gift-box-${playCount}`}>...</motion.div>
  )}
  {state.phase === 'SENTENCE' && (
    <motion.div key={`sentence-${state.sentenceIndex}-${playCount}`}>...</motion.div>
  )}
  {state.phase === 'FINALE' && (
    <motion.div key={`finale-${playCount}`}>...</motion.div>
  )}
  {state.phase === 'PHOTOS' && (
    <motion.div key={`photos-${playCount}`}>...</motion.div>
  )}
</AnimatePresence>
```

### Accessibility: prefers-reduced-motion

When `prefers-reduced-motion: reduce` is active:
- Gift box: instant open (no split animation), skip particle burst
- Sentence revealer: show full sentence immediately (no typewriter), crossfade 100ms
- Photo slideshow: no crossfade, instant swap
- Confetti: reduce to 20 particles, no spread animation
- All transitions: duration 0ms or 100ms max

Implementation: check `window.matchMedia('(prefers-reduced-motion: reduce)')` in ExperienceOrchestrator and pass `reducedMotion` boolean prop to all child components.

## Theme Application

The wish's theme ID determines the dynamic CSS custom properties set on the experience container:

```css
/* Set via inline style on .experience wrapper */
.experience[data-theme="ocean"] {
  --theme-primary: #4d96ff;
  --theme-secondary: #6bcb77;
  --theme-surface: #e8f4f8;
}
```

Theme surface becomes the page background. Theme primary/secondary are used for:
- Gift box lid and ribbon colors
- Glow gradient
- Confetti particle colors
- Sender name color
- Active progress dots

Components read from `--theme-primary`, `--theme-secondary`, `--theme-surface` custom properties (set by data-attribute on the wrapper) rather than directly from the THEMES object at render time. This keeps the CSS-first approach consistent.
