Create the error handling infrastructure in three files, then refactor the service and route layer:

**1. Create `server/lib/errors.js`** (per D-01, D-02, D-04)

Create this new file with ESM exports. It must contain:

- `ValidationError` class extending Error: `this.name = 'ValidationError'`, `this.status = 400`, `this.code = 'VALIDATION_ERROR'`. Constructor takes `message` string.
- `NotFoundError` class extending Error: `this.name = 'NotFoundError'`, `this.status = 404`, `this.code = 'NOT_FOUND'`. Constructor takes `message` with default `'Resource not found'`.
- `DatabaseError` class extending Error: `this.name = 'DatabaseError'`, `this.status = 500`, `this.code = 'DB_ERROR'`. Constructor takes `message` string.
- `asyncHandler(fn)` utility function: returns a middleware that calls `Promise.resolve(fn(req, res, next)).catch(next)`. This solves Express 4's lack of async error catching (Pitfall A in RESEARCH.md).

All exports must use ESM `export` syntax (server/package.json has `"type": "module"`).

**2. Create `server/middleware/errorHandler.js`** (per D-03, D-04)

Create this new file exporting a single `errorHandler(err, req, res, next)` function (4-arg Express error handler signature).

Behavior:
- Logs `[${err.name || 'Error'}] ${err.message}` to console.error
- Reads `err.status` (default 500) and `err.message` (default 'Internal server error') -- per D-03, always surface the real error message, never hide behind generic strings
- Builds response object `{ error: err.message }` -- adds `code: err.code` only if `err.code` exists (per D-01: code is optional)
- Calls `res.status(status).json(response)`

**3. Refactor `server/services/wishService.js`** (per D-05)

Read the file first. Add import at top:
`import { ValidationError, NotFoundError, DatabaseError } from "../lib/errors.js";`

Replace each `throw new Error(...)` with the appropriate custom class:
- Line 28: `throw new Error("senderName, recipientName, and message are required")` -> `throw new ValidationError("senderName, recipientName, and message are required")`
- Line 32: `throw new Error("Message must be 1000 characters or less")` -> `throw new ValidationError("Message must be 1000 characters or less")`
- Line 36: `throw new Error("Maximum 5 photos allowed")` -> `throw new ValidationError("Maximum 5 photos allowed")`
- Line 71-74: `throw new Error("Database error creating wish: ...")` -> `throw new DatabaseError("Database error creating wish: " + dbErr.message + (dbErr.code ? ` (code: ${dbErr.code})` : ""))`
- Line 95: `throw new Error("Database error fetching wish: ...")` -> `throw new DatabaseError("Database error fetching wish: " + dbErr.message)`
- Line 99: `throw new Error("Wish not found: " + id)` -> `throw new NotFoundError("Wish not found: " + id)`
- Line 145: `throw new Error("Wish not found")` -> `throw new NotFoundError("Wish not found")`

Do NOT change any logic, query structure, or return values -- only the error throw types change.

**4. Refactor `server/routes/wishes.js`** (per D-04, D-05)

Read the file first. Replace the entire route file content:

Add import at top:
`import { asyncHandler } from "../lib/errors.js";`

For each route handler, remove the inline try/catch blocks and wrap with `asyncHandler`:

- POST `/` route: Remove try/catch (lines 13-27). Wrap with `asyncHandler(async (req, res) => { ... })`. The handler body is just: `const wish = await createWish(req.body); res.status(201).json(wish);` -- errors propagate to errorHandler via asyncHandler.
- GET `/:id` route: Remove try/catch (lines 31-41). Wrap with `asyncHandler`. Handler: `const wish = await getWish(req.params.id); res.json(wish);`
- POST `/:id/reactions` route: Remove try/catch (lines 44-57). Wrap with `asyncHandler`. Keep the inline validation for missing `emoji` field -- throw `ValidationError("emoji is required")` instead of `return res.status(400).json(...)`. Import `ValidationError` from `../lib/errors.js`. Handler: validate emoji exists, compute delta, call `addReaction`, respond with result.
- GET `/:id/stats` route: Remove try/catch (lines 60-71). Wrap with `asyncHandler`. Handler: `const stats = await getWishStats(req.params.id); res.json(stats);`

The key change: NO route handler has try/catch anymore. ALL errors flow through asyncHandler -> next(err) -> errorHandler middleware.

**5. Wire error handler into `server/index.js`** (per D-04)

Read the file first. Add import after existing imports (after line 6):
`import { errorHandler } from "./middleware/errorHandler.js";`

Add the errorHandler mount AFTER the SPA catch-all (after line 34, before `const PORT`):
```
// Centralized error handler — must be LAST middleware (RESEARCH.md Pitfall B)
app.use(errorHandler);
```

This is critical: the error handler MUST be the very last `app.use()` call. Express matches middleware in registration order. If registered before routes, it intercepts all requests.

**No schema changes.** The Prisma schema, WAL mode config, and database file already exist and are correct. No `npx prisma db push` or `npx prisma generate` needed for this phase.
